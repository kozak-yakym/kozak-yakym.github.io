  • System Development for Android Workshops - Linux Kernel Module

    About In 2017, I completed the "System Development for Android Workshops" course at GlobalLogic Ukraine, focusing on Linux Kernel development. This module serves as an educational tool for understanding kernel module development, particularly in handling hardware interrupts and communicating with user-space applications through debugfs. Check it out on GitLab.
  • Challenge The challenge was to develop a Linux kernel module that could handle interrupts, manage data with a custom circular buffer, and interface with user-space through debugfs, all while ensuring efficient and reliable operation in the kernel environment.
  • Solution Description The solution involved creating a kernel module that demonstrates various kernel programming concepts such as interrupt handling, tasklets, circular buffers, and the debugfs interface. The module is designed to be a practical educational tool for those new to kernel development.
  • My Contribution I was responsible for developing the interrupt handling logic, implementing the circular buffer, and setting up the debugfs interface. This included writing automation scripts for module compilation and testing with QEMU, enhancing the module's functionality to include a history buffer.
  • Current Status The project is currently in a maintenance phase, with plans to review and refactor the driver's code, implement CI/CD for automated testing and deployment, and write unit tests to ensure code quality and reliability.
  • Technologies, Tools, and Languages Developed in C, using GCC for cross compilation, Docker, Shell scripts, Linux Kernel, Ring buffer, Linux Kernel interrupts, Tasklets.

  • Weather Service Project

    About The Weather Service project is a Node.js-based system designed to collect weather data, date, and time, and then broadcast this information to an MQTT broker. This service acts as a bridge, enabling the integration of weather data and time into systems that may not directly support such features, particularly in environments with older technology constraints. Check it out on GitLab.
  • Challenge The inception of this project was driven by a need to overcome specific limitations encountered with displaying dashboard outputs on older tablets, specifically those running Android 4.4. These devices were unable to support the HomeAssistant application and other third-party applications for interfacing with the HomeAssistant server.
  • Solution Description The solution was to extend MQTT functionality to include weather data and time by finding a compatible application, MQTT-Dash, that worked on these older devices, thus enabling the desired dashboard functionality.
  • My Contribution This project marks a significant personal milestone in the use of Node.js for practical, real-world applications, serving as a critical component in the smart home setup, ensuring that weather data and time are readily available for display on older tablet devices.
  • Current Status The Weather Service project has been operational for over three years, continuing to serve as a critical component in the smart home setup, enhancing the functionality and user experience of the smart home dashboards.
  • Technologies, Tools, and Languages Node.js, MQTT, MQTT-Dash for dashboard functionality, Weather service API.

  • Radio Frequency Data Collecting System Project

    About The Radio Frequency Data Collecting System (RF DCS) is a project initiated in 2015, focusing on the design and implementation of a system for monitoring and collecting data from various sensors, primarily temperature, using RF communication. It has evolved to include data visualization through Python. Check it out on GitLab.
  • Challenge The main challenge was to create a reliable and efficient system for temperature monitoring and data collection in various environments, overcoming limitations of existing solutions in terms of adaptability and precision.
  • Solution Description The solution involves a PC application written in Python, interfacing with a virtual COM port to communicate with a radio module hub. This hub, built around an ATmega8 microcontroller and equipped with an RFM70 radio module, orchestrates data collection from radio sensor nodes.
  • My Contribution My contribution lies in advancing RF-based data collection systems by developing a modular, scalable architecture that supports a wide range of sensors and provides detailed data visualization capabilities, enhancing the understanding and monitoring of environmental conditions.
  • Current Status For now, I don't plan to maintain this code and leave it as it is, as I wrote it in 2015. Subsequent projects leveraging this system's architecture planned to include a fuel burnout notification device and a solid fuel boiler control system, each undergoing its unique development process.
  • Technologies, Tools, and Languages Uses ATmega8 and ATTiny13 microcontrollers, RFM70 radio module, C, Python, and virtual COM port communications.

  • Heating System Data Visualization Tool

    About A tool designed for visualizing data from a heating system equipped with a wood-burning boiler. Initially created to assist in understanding the operational dynamics of a boiler system through the visualization of sensor data collected via a custom-built RF sensor acquisition system. Check it out on GitHub.
  • Challenge The primary challenge was to devise a method that allowed for the effective monitoring and analysis of a wood-burning boiler heating system's performance using sensor data. This endeavor required surmounting technical hurdles, notably the necessity for an enhanced data visualization strategy. My aim was to develop a graphical representation that supported detailed examination through zooming capabilities and the ability to isolate a single curve for focused analysis. Additionally, I sought to implement a feature that would enable the comparison of two distinct curves, facilitating the identification of potential correlations.
  • Solution Description The solution involved the development of a PC application in Python for data visualization. This application processes data collected from various sensors placed throughout the heating system and visualizes it to provide insights into the system's performance. External data, such as outdoor temperature from a weather server API, was also integrated to compensate for sensor limitations.
  • My Contribution My contribution extends to the field of heating system monitoring by providing a detailed visualization tool that aids in the analysis of boiler efficiency and environmental conditions. This project showcases the application of Python for data analysis and visualization, enhancing the understanding of wood-burning boiler systems.
  • Current Status Currently, the project is under development with plans for further improvements in modularity, code refactoring, and documentation. Future enhancements include adding execution parameters to the Python script, improving the data collection system, and exploring the potential for a live view feature to monitor the system in real-time.
  • Technologies, Tools, and Languages Developed using Python, with matplotlib visualization library.
